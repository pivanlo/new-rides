About this implementation

  • Implemented with React, Node.js and the Express web framework.
  • Uses Server-Sent Events (SSE) for real time communication.
  • Uses a polyfill to support browsers where "EventSource" is not available.
  • Handles connection recovery by sending missed event messages.
  • Handles connection closure by the client (triggered by the "Stop updates" button).
The interesting files are:
  • index.js (starts the server and routes client requests)
  • sse.js (module for sending SSE messages)
  • messages.js (the React component)
